# 用户调研报告

## 调研概述

**产品名称**: 校园通 (CampusHub)
**产品定位**: 校园本地信息聚合平台
**核心场景**: 校招内推、租房找房、二手交易、生活信息交流、考研考公上岸
**调研目标**: 理解目标用户群体特征、核心需求、使用场景及痛点，为产品设计提供依据

## 目标用户画像

### 用户群体总体特征

| 维度 | 描述 |
|------|------|
| 年龄范围 | 18-28 岁 |
| 身份构成 | 本科生、硕士研究生、博士研究生、应届毕业生 |
| 地域特征 | 华北理工大学（曹妃甸校区）；周边主流生活服务平台（大众点评、58同城等）覆盖率低，校园内部信息流转更为关键 |
| 技术素养 | 高：熟练使用移动互联网产品，对信息质量和社交传播敏感 |
| 消费特征 | 预算有限但消费意愿不低；价格敏感；注重性价比 |
| 社交特征 | 强校园归属感，信任同校/同城学生；信息获取高度依赖社交网络和垂直社区 |

## 场景一：校招内推

### 子模块结构

| 子模块 | 核心内容 | 筛选维度 |
|--------|----------|----------|
| 实习 | 实习岗位信息、实习面经、实习体验分享 | 专业、学院、公司 |
| 找工作经验 | 校招时间线、面试真题、offer 选择建议 | 专业、行业、公司 |
| 内推 | 校友内推通道、内推码、内推进度跟踪 | 学院、公司、岗位 |
| 技能学习 | 简历优化、笔试刷题、面试技巧、行业入门 | 专业、技能方向 |

> **核心筛选机制**：支持按**专业**（计算机科学、电子信息、金融等）和**学院**（信息学院、经管学院、机械学院等）双维度筛选信息，让不同专业背景的学生看到针对性的内容。

### 1. 用户画像

#### 人物 A：大四应届生 "小李"
- 背景：华北理工大学计算机专业，GPA 中等偏上，有一段中小厂实习经历
- 求职状态：海投策略，同时在看春招和秋招补录
- 信息获取渠道：牛客网、Boss直聘、学校就业网、学长学姐微信群
- 日常行为：每天刷牛客面经、检查邮箱、在各个群里蹲内推码

#### 人物 B：研二学生 "小王"
- 背景：本校电子信息硕士，研究方向与工业界对口
- 求职状态：精准投递，目标头部大厂核心岗位
- 信息获取渠道：实验室师兄内推、脉脉、LinkedIn、学术会议
- 日常行为：关注特定公司的招聘时间线，定向联系目标部门的校友

### 2. 核心需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 内推信息获取 | P0 | 快速找到可靠的内推渠道和内推码 |
| 面经/笔试经验 | P0 | 获取目标公司/岗位的真实面试真题和通过经验 |
| 招聘信息聚合 | P1 | 一站式查看多公司校招时间线、截止日期 |
| 校友内推连接 | P1 | 找到同校/同专业在目标公司就职的校友 |
| 薪资/offer 对比 | P2 | 了解不同公司 offer 的薪资水平和工作体验 |
| 求职社区交流 | P2 | 与同届求职者交流进度、共享信息 |

### 3. 痛点分析

1. **信息分散**: 招聘信息散落在各家官网、牛客、Boss直聘、微信群、公众号，需要同时关注 10+ 信息源，容易遗漏关键时间节点
2. **内推码质量参差**: 群里的内推码真假难辨，部分内推人只是为了赚内推奖励，不会真正跟进；真正有效的内推渠道隐蔽且难获取
3. **信息过载与焦虑**: 海量面经和 offer 对比帖造成横向比较压力，缺乏按个人背景筛选的精准信息
4. **校友资源利用不足**: 大多数学生不知道本校有哪些校友在目标公司，缺乏系统的校友内推对接机制
5. **校园招聘会信息滞后**: 学校就业网更新不及时，企业宣讲会信息分散，缺少手机端及时提醒

### 4. 行为特征

- **高频使用期**: 秋招 8-11 月、春招 2-4 月为使用高峰
- **使用时段**: 晚间和周末集中浏览，碎片时间（课间、排队）快速检查更新
- **设备偏好**: 手机端为主（刷信息、聊天交流），PC 端为辅（投递简历、做笔试）
- **决策周期**: 从看到岗位到投递通常在 1-3 天内
- **社交驱动**: 高度依赖熟人推荐和校友网络建立信任

## 场景二：租房找房

### 子模块结构

| 子模块 | 核心内容 |
|--------|----------|
| 租房 | 找房源、看房攻略、合同签约、中介识别 |
| 出租 | 房东/转租人发布房源、毕业生退租房源流转 |
| 找合租室友 | 室友匹配（预算/作息/性别/吸烟等）、合租经验分享 |

### 1. 用户画像

#### 人物 C：大三升研零 "小张"
- 背景：保研本校，需要在暑假期间找到研究生阶段的校外住房
- 租房状态：第一次租房，对租房流程、合同条款不了解
- 信息获取渠道：学校论坛/贴吧、豆瓣租房小组、贝壳/自如 App、师兄师姐推荐

#### 人物 D：毕业入职 "小陈"
- 背景：拿到 offer 即将入职，需要在公司附近租房
- 租房状态：时间紧迫（2-4 周内必须入住），预算有限，可能需合租
- 信息获取渠道：公司新人群、自如/贝壳、58同城、小红书

### 2. 核心需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 真实房源信息 | P0 | 避开虚假房源和中介钓鱼帖，看到真实有效的出租信息 |
| 同校/同城合租匹配 | P0 | 找到靠谱的校友或同龄人合租，降低租金和生活摩擦 |
| 价格透明度 | P1 | 了解区域真实租金水平，避免被宰 |
| 租房避坑指南 | P1 | 合同注意事项、押金退还、二房东识别等经验 |
| 地理信息辅助 | P1 | 通勤时间估算、周边生活配套（超市、医院、地铁） |
| 短期/过渡租房 | P2 | 实习期短租、考研季短租等灵活需求 |

### 3. 痛点分析

1. **虚假信息泛滥**: 58同城、安居客等平台大量虚假低价房源引流，真正有效的房源被淹没
2. **中介费高昂**: 通过中介找房通常需支付一个月租金作为中介费，对学生负担重
3. **信息不对称**: 不熟悉城市各区域真实租金水平，容易被房东或中介高价出租
4. **合租匹配困难**: 现有合租平台（如豆瓣小组）信息混乱，缺乏结构化筛选（预算、作息、性别等）
5. **校园墙依赖严重**: 在曹妃甸校区周边，房东不习惯使用58同城、贝壳等主流平台，房源信息几乎仅通过校园墙（QQ空间/微信号）转发扩散。学生找房需关注多个校园墙账号，信息格式不统一、过期信息无法标记、翻找历史消息极为低效
6. **校友资源闲置**: 每年大量毕业生退租、新生找房，但这些信息在校内缺乏有效流转渠道
7. **时间窗口紧张**: 毕业季/开学季找房时间紧迫，决策压力大

### 4. 行为特征

- **高频使用期**: 5-6 月（毕业季退租）、8-9 月（开学季找房）
- **使用时段**: 全天候，周末集中看房
- **设备偏好**: 手机端浏览房源、沟通；PC 端查看地图和详细信息
- **决策周期**: 从看到房源到决定通常在 3-7 天内
- **信任锚点**: 房东/转租人的学校身份验证是建立信任的关键
- **渠道特色**: 郊区校园周边房东高度依赖校园墙传播房源，形成"房东 → 校园墙 → 学生群/朋友圈 → 个人"的非标准化信息链路

## 场景三：二手交易

### 子模块结构

| 子模块 | 核心内容 | 典型物品 |
|--------|----------|----------|
| 书本 | 教材买卖、考研/考公资料、课外书流转 | 《数据结构》、肖秀荣全家桶、四六级真题 |
| 生活用品 | 电子电器、家具、运动器材、日用品 | 台灯、电风扇、人体工学椅、哑铃 |
| 非实体交易 | 课程名额转让、会员共享、跑腿服务 | 驾校名额、网盘会员、健身房卡转让 |

### 1. 用户画像

#### 人物 E：毕业离校 "小赵"
- 背景：大四即将毕业，需要在离校前 1-2 周内处理大量闲置
- 核心物品：教材/考研资料、电子产品（台灯、电扇、路由器）、生活用品（衣架、收纳箱）、运动器材
- 痛点：物品多且杂，逐一拍照上架闲鱼太耗时；大件物品邮寄成本高，倾向本地面交

#### 人物 F：入学新生 "小周"
- 背景：大一/研一新生，需要低成本购置学习和生活用品
- 核心物品：教材（正版太贵）、台灯/风扇等小电器、自行车/电动车
- 痛点：不熟悉学校和周边环境，不知道去哪里淘二手；担心买到的物品有暗病

### 2. 核心需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 快速发布 | P0 | 毕业季大量物品需快速上架，操作越简单越好 |
| 校内/附近交易 | P0 | 面交可省运费，校园内交易安全便捷 |
| 物品分类浏览 | P1 | 按类别（教材、电子、生活等）高效筛选 |
| 卖家/买家信用 | P1 | 校园身份背书，降低交易风险和纠纷 |
| 价格参考 | P2 | 了解同类物品的二手行情定价 |
| 批量处理 | P2 | 打包出售、免费赠送等场景支持 |

### 3. 痛点分析

1. **平台不匹配**: 闲鱼面向全社会，信息流中充斥着专业卖家和引流广告；学生身份无法得到有效验证和突显
2. **时间窗口短**: 毕业季 1-2 周的出清窗口极短，传统平台发布流程繁琐，效率低
3. **运费制约**: 大件物品（桌椅、床垫）跨城邮寄成本高，需要就近交易，但现有平台地理位置筛选不够精准
4. **信任缺失**: 匿名化交易中买卖双方均存在不信任感，物品描述与实物不符的情况频发
5. **信息孤岛**: 校内二手交易主要通过 QQ 群、微信群进行，信息分散不互通，无法有效匹配
6. **教材循环缺乏体系**: 每学期同样的教材在新生和老生之间流转，但目前缺乏高效的匹配机制

### 4. 行为特征

- **高频使用期**: 6-7 月（毕业季出清）、9 月（新生入学采购）、每学期末
- **使用时段**: 晚间集中上架或浏览
- **设备偏好**: 手机端为主，拍照+发布一站式完成
- **决策周期**: 价格敏感，货比三家，通常在 1-3 天内完成
- **交易偏好**: 校内面交 > 同城面交 > 邮寄

## 场景四：生活信息交流

### 子模块结构

| 子模块 | 核心内容 | 内容形态 |
|--------|----------|----------|
| 表白 | 匿名/实名表白墙、恋爱求助、情感树洞 | 短文/匿名帖 |
| 吐槽 | 校园吐槽、课程吐槽、食堂吐槽、生活趣事 | 短文/带图帖 |
| 照片分享 | 校园摄影、日常随拍、毕业照分享、宠物 | 图片为主 |
| 美食推荐 | 食堂档口评测、外卖推荐、周边餐厅探店 | 评测/打分+图片 |

### 1. 用户画像

#### 人物 G：大二在校生 "小刘"
- 背景：普通高校大二学生，对校园生活充满探索欲
- 日常状态：关注食堂新窗口、想知道哪家外卖好吃、选课纠结中、想找一起运动的搭子
- 信息获取渠道：学校超话、小红书、微信群、口口相传
- 痛点：信息零散分布在各个平台，没有一个集中的校园生活信息枢纽

#### 人物 H：研一新生 "小林"
- 背景：考研上岸，对新环境完全陌生
- 日常状态：不知道哪个食堂好吃、不了解周边生活配套、想认识新朋友、需要办理各种校园手续
- 信息获取渠道：实验室同门、新生群、小红书搜索"XX大学攻略"
- 痛点：缺乏系统化的校园生活指南，每一个小问题都要到处问人

### 2. 核心需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 校园攻略/评测 | P0 | 食堂档口、周边美食、打印店、快递点等生活设施的评测和推荐 |
| 搭子匹配 | P1 | 找饭搭子、运动搭子、学习搭子，降低校园社交门槛 |
| 选课/教师评价 | P1 | 课程难度、给分情况、教师授课风格的真实评价 |
| 活动信息聚合 | P1 | 社团招新、讲座、比赛、演出等校园活动一站式查看 |
| 求助/问答社区 | P2 | 校园生活相关问题快速得到校友解答 |
| 技能交换/互助 | P2 | 修电脑、借东西、代取快递等邻里式互助 |

### 3. 痛点分析

1. **信息碎片化且无平台依托**: 食堂档口、周边外卖、街边小馆到底哪家好吃——这些"小微信息"散落在微信群和口口相传中。在曹妃甸校区周边，商家几乎不入驻大众点评，食堂档口更无任何线上评分——学生完全依赖熟人推荐，但现有渠道（微信群/QQ群）无法做到信息的结构化沉淀和检索
2. **新生信息鸿沟**: 每年新生入学都要重新踩坑（难吃的食堂、坑人的打印店），老生的经验无法有效沉淀和传承
3. **选课盲目性**: 选课系统只有课程名称和教师名，缺乏真实的课程评价和给分情况，全靠找学长学姐私下打听
4. **社交通道单一**: 校园社交依赖 QQ 群和年级大群，缺乏兴趣导向的轻量社交场景，"找搭子"成为刚需但缺少好用的工具
5. **活动信息闭塞**: 很多质量不错的校园讲座、演出、比赛因为宣传不到位而参与度低，想参加的人又不知道消息
6. **求助效率低下**: 在 500 人大群里问一个问题容易被刷掉，私聊又不知道找谁，缺少高效、低社交压力的求助渠道
7. **校园墙信息混乱不可检索**: 校园墙（QQ空间/微信）是郊区校园信息流转的事实基础设施，但每天几十条投稿混在一起——表白、吐槽、租房、二手、失物招领、美食推荐——信息无法分类筛选，24小时后即被新内容淹没，优质内容（如详细的食堂评测、租房攻略）无法沉淀

### 4. 行为特征

- **高频使用期**: 全年稳定使用，开学季（9-10 月、2-3 月）为使用高峰
- **使用时段**: 碎片时间频繁浏览（课间、排队、睡前），内容消费型场景
- **设备偏好**: 几乎全部手机端，浏览/发布都追求轻量化
- **内容生产**: 大部分用户是消费者，少数活跃用户生产内容（评测、攻略）
- **信息类型**: 图文为主，追求真实感而非精美排版；评分/推荐类信息受信任度最高

## 场景五：考研考公上岸信息交流

### 子模块结构

| 子模块 | 核心内容 |
|--------|----------|
| 考研 | 院校/专业选择、各科备考经验、资料共享、复试调剂、导师选择 |
| 考公 | 岗位选择策略、行测/申论经验、面试心得、事业单位/选调生 |

### 1. 用户画像

#### 人物 I：大三备考 "小孙"
- 背景：本校学生，目标 211 院校计算机专业，大三下学期开始备考
- 备考状态：焦虑但努力，每天都在找资料、看经验帖、跟网课
- 信息获取渠道：知乎经验帖、B站网课、微信公众号、考研 QQ 群、学长学姐一对一咨询
- 核心困惑：不知道该用什么资料、不知道目标院校真实难度、不知道自己的进度是否正常

#### 人物 J：二战考生 "小杨"
- 背景：一战差 5 分未上岸，毕业后租房全职备考
- 备考状态：压力极大，经济紧张（无收入），需要高性价比的复习资源
- 信息获取渠道：考研论坛、上岸学长经验、线下自习室
- 核心困惑：需要针对性的薄弱环节突破方案、需要二手资料省钱、需要同城考友互相监督

#### 人物 K：考公应届生 "小吴"
- 背景：普通本科法学专业，目标省会城市公务员
- 备考状态：同时在准备省考和事业单位联考，时间分配焦虑
- 信息获取渠道：粉笔 App、华图/中公公众号、知乎、党校学长
- 核心困惑：岗位选择策略、笔试面试时间线、不同地区的待遇和竞争比

### 2. 核心需求

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 上岸经验分享 | P0 | 目标院校/岗位的真实备考经验、时间规划、资料推荐 |
| 备考资料交流 | P0 | 二手教材/笔记、网课资源互换、真题资料的流转 |
| 同校/同城考友匹配 | P1 | 找到目标相同或相近的备考搭子，互相监督打卡 |
| 院校/岗位情报 | P1 | 报录比、分数线趋势、复试风格等关键决策信息 |
| 备考进度对标 | P2 | 了解同届考生的复习进度，评估自身备考状态 |
| 自习室/备考环境 | P2 | 学校周边自习室、图书馆座位信息、安静租房推荐 |

### 3. 痛点分析

1. **信息真假难辨**: 考研/考公培训机构的营销内容充斥网络，"上岸经验"中夹带软广，真实参考价值大打折扣
2. **精准情报获取难**: 特定院校、特定导师的信息极度隐蔽，通常只在 QQ 群或私聊中流传，公开渠道找不到
3. **资料成本高**: 正版考研资料（如肖秀荣全家桶、各机构网课）价格不菲，二手资料的流转缺乏高效渠道
4. **备考孤独感**: 全职备考（二战或在职考）容易产生社交隔离和心理压力，缺少同路人互相支持
5. **择校/选岗试错成本高**: 考研报错学校、考公报错岗位可能直接导致落榜，决策阶段的信息质量至关重要
6. **机构营销噪音**: 社交媒体上"3 个月逆袭上岸"之类的鸡汤帖子制造无效焦虑，缺乏基于真实数据的理性参考

### 4. 行为特征

- **高频使用期**: 考研 3-6 月开始择校准备、7-12 月密集备考、次年 2-4 月复试调剂；考公 10-11 月国考、3-4 月省考
- **使用时段**: 集中复习时段外的碎片时间刷信息，晚上复盘时段深入浏览经验帖
- **设备偏好**: 手机端刷帖子交流，平板/PC 端看网课学资料
- **决策周期**: 择校/选岗阶段决策周期较长（1-4 周），资料购买决策快（1-2 天）
- **社交特征**: 高度功利导向的社交——找考友的首要目的是互相督促和信息交换，伴随强烈的竞争意识

## 跨场景用户洞察

### 场景交叉

五个场景覆盖了大学生从入学到毕业的全生命周期需求：

| 用户阶段 | 主要场景 | 次要场景 |
|----------|----------|----------|
| 大一/研一新生 | 生活信息交流、二手交易（买入） | - |
| 大二/研一 | 生活信息交流、二手交易（买卖） | 考研考公（开始关注） |
| 大三/研二上 | 考研考公上岸（择校/选岗） | 生活信息交流、二手交易 |
| 大三/研二下 | 校招内推（实习/秋招）、考研考公（备考） | 二手交易（卖出） |
| 大四/研三 | 校招内推（求职）、租房、考研考公（冲刺/调剂） | 二手交易（大量卖出） |
| 毕业入职 | 租房 | 校招内推（社招过渡） |

### 用户生命周期

```
入学 → 生活信息消费 → 二手循环买卖 → 考研考公/关注求职 → 深度求职 + 租房 → 毕业出清
  |                                                                              |
  +------------- 4 年（本科）或 6-7 年（本硕）的用户生命周期 ---------------------+
```

### 共性需求提炼

1. **校园身份验证**: 所有场景都依赖于"我是本校学生"的身份背书来建立信任
2. **信息真实性**: 用户对虚假信息容忍度极低，真实性是留存的关键
3. **轻量化操作**: 学生时间碎片化，需要极简的发布和浏览流程
4. **校友网络价值**: 同校/同城学生之间的信息流转天然具有信任基础
5. **季节性波动**: 五个场景都有明显的季节性高峰，需要产品节奏与之匹配
6. **信息结构化**: 从食堂测评到考研经验，用户需要的是结构化、可检索的信息，而不是聊天记录式的碎片内容

## 用户需求优先级矩阵

|   | 高频 | 低频 |
|---|------|------|
| **高痛** | 内推信息获取、真实房源、快速发布二手、上岸经验分享、校园生活攻略 | 校友内推对接、合租匹配、批量出清、备考进度对标 |
| **低痛** | 招聘信息聚合、物品分类浏览、选课评价、考友匹配 | 薪资对比、价格参考、技能交换、自习室信息 |

- **先做**: 高频+高痛 = 内推信息获取、真实房源、快速发布、上岸经验、校园攻略 → MVP 核心
- **第二梯队**: 高频+低痛 = 信息聚合、分类浏览、选课评价 → 体验优化
- **第三梯队**: 低频+高痛 = 校友对接、合租匹配、搭子匹配 → 差异化竞争力
- **最后**: 低频+低痛 = 辅助工具 → 锦上添花

## 调研方法说明

本调研基于以下方法综合形成：

1. **桌面研究**: 分析知乎、小红书、牛客网、豆瓣等平台大学生相关讨论，梳理各场景下的行为模式和痛点
2. **竞品体验**: 深度使用闲鱼、牛客、贝壳找房、豆瓣租房小组、大众点评、小红书、粉笔等竞品，记录功能优劣势
3. **公开数据参考**: 参考高校就业报告、租房市场白皮书、二手交易行业报告、教育部考研报考数据中的大学生相关数据
4. **逻辑推演**: 基于团队成员自身校园经历，推演用户行为路径和需求优先级

> 注：本阶段为项目初期的桌面调研，后续建议补充定量问卷和定性用户访谈以验证假设。
